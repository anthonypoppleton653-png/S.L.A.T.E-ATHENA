Example: CI/CD Pipeline

name: SLATE CI/CD
on: [push, pull_request]
jobs:
  build:
    runs-on: [self-hosted, slate]
    steps:
      - uses: actions/checkout@v4
      - name: Setup Python
        run: |
          "$env:GITHUB_WORKSPACE\.venv\Scripts" |
            Out-File -Append $env:GITHUB_PATH
      - name: Lint
        run: black --check slate/ agents/ tests/
      - name: Test
        run: pytest tests/ --tb=short -q
      - name: Security Scan
        run: python slate/action_guard.py --scan
      - name: GPU Validation
        run: python slate/slate_hardware_optimizer.py
      - name: Build Docker
        run: docker build -t slate:latest .
      - name: Deploy to K8s
        if: github.ref == 'refs/heads/main'
        run: python slate/slate_k8s_deploy.py --deploy
permissions: { contents: read, packages: write }
timeout-minutes: 15

Architectural Principles

  • Local-First: All operations bind to 127.0.0.1 — zero external network exposure
  • Hardware-Aware: Dual-GPU load balancing with model-to-GPU affinity
  • Container-First: K8s primary, Docker secondary, no local fallback in extension
  • Self-Healing: Autonomous loops detect and recover from failures
  • Adaptive: K8s-driven instruction layer adjusts behavior based on system state

ML Equations

Cross-Entropy Loss: L = -∑ yi log(pi)

Adam Optimizer: mt = β1 mt-1 + (1-β1) gt

Bias Correction: m̂t = mt / (1 - β1t)

Parameter Update: θt = θt-1 - η · m̂t / (√v̂t + ε)

Cosine Similarity (Embeddings): sim(A,B) = (A · B) / (|A| · |B|)

Inference Throughput: T = tokens / (latency_ms / 1000)

Security Principles

  • Local-Only: All bindings to 127.0.0.1 — never 0.0.0.0
  • No External Telemetry: ChromaDB telemetry disabled
  • Protected Files: Fork security gate blocks workflow / CODEOWNERS modifications
  • Credential Isolation: Git credential manager, no stored tokens
  • Container Isolation: Commands execute in K8s/Docker, not host

2026 Research Integration

GitHub Actions Best Practices

  • Custom autoscaling runners with GPU affinity labels
  • Reusable workflows with clear ownership and documentation
  • Secrets management via credential managers (not env vars)
  • Least privilege permissions, action pinning to commit SHAs
  • Parallel job execution with dependency graph optimization
  • Timeout enforcement (15min default, 30min GPU jobs)

Docker Best Practices

  • Multi-stage builds: CUDA 12.8 base → Python deps → App layer
  • Non-root user execution (USER slate)
  • .dockerignore for build context optimization
  • Resource limits (deploy.resources in compose)
  • GPU passthrough via NVIDIA Container Toolkit
  • CPU-only fallback image (Dockerfile.cpu)

Prompt Engineering Best Practices

  • Structured inputs with clear role definitions (system/user/assistant)
  • Chain-of-Thought (CoT) for complex reasoning tasks
  • Few-shot examples for consistent output formatting
  • Self-criticism prompting for quality validation
  • Forced output schemas (JSON/YAML) to reduce hallucinations
  • Agent routing via intent classification (slate-fast model)

Adaptive Instructions (K8s-Driven)

  • Instructions dynamically generated from system state
  • K8s ConfigMap slate-instructions as source of truth
  • 6 operating modes: NORMAL, DEGRADED, MAINTENANCE, AUTONOMOUS, EMERGENCY, DEVELOPMENT
  • CronJob syncs instructions every 5 minutes
  • Fallback to local state if K8s unreachable

Communication & Coordination

Multi-participant chaining: @slate for system ops, @vscode for editor ops. Structured prompt patterns with feedback loops.

Escalation Order

1. Participant Tool   (e.g., slate_workflow, slate_specKit)

3. Subagent Delegation (runSubagent with 'slate')
